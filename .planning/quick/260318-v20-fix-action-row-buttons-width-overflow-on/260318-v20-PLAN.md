# Quick Plan 260318-v20: Fix Action Row Button Width

## Goal

The three action row buttons (Register Mother, Clinic Visit, Add Vaccination) are not constrained to the 393px container width on iPhone 16 Pro. They overflow because each button uses natural intrinsic width (`whiteSpace: nowrap`, no `flex: 1`).

## Root Cause

In `app/dashboard/page.tsx`, the action row container is `flex items-center justify-center` with a gap, but each `<button>` has no `flex: 1` — so buttons use natural content width and together they exceed the available row width.

## Fix

**Task 1: Add `flex: 1` + `justifyContent: 'center'` to each of the 3 action row buttons**

- File: `app/dashboard/page.tsx`
- Change: Add `flex: 1` and `justifyContent: 'center'` to the inline style of each button in the Action Row section (lines ~397–469)
- Also add `width: '100%'` to the action row container div to ensure it fills the 393px column

## Verification

- Buttons fill the full row equally at 393px width
- No overflow, no wrapping, no clipping
- Wireframe matches: three equal-width pills side by side
